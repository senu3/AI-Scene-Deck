# Docs Guide

**目的**: 新規ドキュメント追加時の配置ルールと最小要件を定義する。
**適用範囲**: `docs/` 配下の Markdown。
**関連ファイル**: `docs/INDEX.md`。
**更新頻度**: 中。

## 追加ルール
- 入口は必ず `docs/INDEX.md` から辿れるようにする。
- 既存の正本は `docs/references/DOMAIN.md` と `docs/references/MAPPING.md`。
- 重複説明は避け、必要ならリンク参照にする。

## 1ファイルの必須項目
- 目的
- 適用範囲
- 関連ファイル
- 更新頻度（高/中/低）
- 未確定事項がある場合は「TODO/仮/未決」を明記

## 分類ルール
- `docs/references/` 用語・対応表などの正本
- `docs/guides/` 実装/運用のガイド
- `docs/ui/` UI設計・コンポーネント・配色
- `docs/notes/` 一時メモや設計途中のノート

## 記述ルール
- 仕様/設計/実装ノートが混ざる場合は章を分ける。
- 参照先（ソースファイル）は必ず記載。
- 長くなる場合は分割する（2〜4画面が目安）。
